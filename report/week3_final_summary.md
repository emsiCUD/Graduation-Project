# Week 3 — Final Summary (Traditional ML Champion)

**Dataset:** ViHSD (Luu et al., 2021) — 3-class Vietnamese hate-speech detection
(0 = CLEAN, 1 = OFFENSIVE, 2 = HATE). Train/Dev/Test = 24 048 / 2 672 / 6 680.
Primary metric: **macro F1** (chosen because of 12 : 1 class imbalance).

---

## 5. Insights for the thesis

1. **Char n-grams >> hyper-parameter tuning.**
   GridSearch on LR word-features alone added only **+0.0009 F1_macro**.
   Adding char-level TF-IDF (3-5) gave **+0.0369**, a 41× larger gain.
   → For Vietnamese toxic comments, **feature engineering matters more than
   regularisation strength** because users obfuscate words with character
   substitutions (e.g. *đm*, *vl*, *clmm*, repeated chars).

2. **Threshold tuning was unnecessary** for this configuration.
   With `class_weight='balanced'` already correcting the prior, the optimal
   bias multipliers are `(off=1.0, hate=1.0)` for both F1_macro and
   F1_OFFENSIVE objectives. Threshold tuning becomes useful only when the
   classifier is **not** class-weight-corrected.

3. **Champion (0.6194 test F1_macro) is on par with the ViHSD paper.**
   The original ViHSD paper reports best traditional-ML macro-F1 around
   ~0.61 (Luu et al., 2021, Table 5). Our pipeline matches that without any
   external lexicon or hand-crafted features — purely via char n-grams +
   class balancing.

---

## 6. Limitations

- **F1_OFFENSIVE = 0.396** is still the weak spot. The class boundary between
  CLEAN/OFFENSIVE/HATE is genuinely ambiguous for short Vietnamese comments,
  and TF-IDF cannot capture context (sarcasm, target of the offence).
- **Bag-of-features ignores word order and dependency context**, which limits
  HATE detection (often relies on phrase-level cues like *"bọn X là …"*).
- No external knowledge (slang dictionary beyond `teencode.json`, dialect
  resources, demographic targets).
